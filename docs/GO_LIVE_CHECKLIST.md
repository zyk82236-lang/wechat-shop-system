# 上线检查清单（Go-Live Checklist）

## P0：上线前必须完成

- 真实微信支付参数与证书配置完成（`WECHAT_*` 全量）。
- 公网 `HTTPS` 域名可访问，`/payments/wechat/notify` 回调可达。
- `DATABASE_URL`、`REDIS_URL` 指向生产实例并验证连接稳定。
- 完整执行 `docs/RELEASE_RUNBOOK.md` 第 3 节全链路验收。
- 至少一次真实支付闭环验证：
  - 下单
  - 支付成功
  - 回调入账
  - 后台发货
  - 用户确认收货

## P1：上线后 1-2 周补齐

- 监控告警：
  - API 5xx 比例
  - 支付回调失败率
  - MySQL 慢查询
  - Redis 可用性与延迟
- 自动化回归（e2e）脚本，覆盖下单支付主链路。
- 管理后台权限细化（按模块与操作粒度）。
- 审计日志（订单状态、物流更新、商品上下架等关键操作）。

## P2：后续迭代优化

- 图片上传改造为对象存储（替代手填 URL）。
- 促销能力（优惠券、满减、活动价）。
- 退款/售后流程完善。
- 数据分析增强（转化率、复购、漏斗看板）。

## 使用方式

- 每次发布前对照 P0 逐项打勾。
- 发布后一个迭代内完成 P1。
- 按业务优先级排期 P2。
